## 1. 引言

### 1.1 编写目的

本文档描述智测心理测试系统的总体设计方案，包括系统架构、模块划分、技术选型等，为详细设计和开发实现提供指导。

### 1.2 背景

本系统基于若依（RuoYi）v3.9.0快速开发框架，采用前后端分离架构，旨在为用户提供便捷的心理测试服务。系统需支持在线测试、自动评分、结果分析等核心功能。

### 1.3 定义

| 术语 | 定义 |
|-----|------|
| 前后端分离 | 前端和后端独立开发，通过API接口通信 |
| RESTful API | 符合REST风格的API接口 |
| JWT | JSON Web Token，用于身份认证 |
| RBAC | 基于角色的访问控制 |

## 2. 总体设计

### 2.1 需求规定

本系统需满足以下核心需求：

1. **功能需求**
   - 用户认证与授权
   - 人格测试功能
   - 题目管理功能
   - 结果分析功能
   - 系统管理功能

2. **性能需求**
   - 响应时间 < 2秒
   - 支持1000+并发用户
   - 系统可用性 ≥ 99.5%

3. **安全需求**
   - 用户数据加密存储
   - 操作日志记录
   - 权限严格控制

### 2.2 运行环境

**服务器环境：**
- 操作系统：CentOS 7+ / Ubuntu 18.04+
- JDK：1.8+
- MySQL：8.0+
- Redis：5.0+
- Nginx：1.18+

**客户端环境：**
- 浏览器：Chrome 90+ / Firefox 88+ / Edge 90+ / Safari 14+

### 2.3 基本设计概念和流程

#### 2.3.1 设计原则

1. **高内聚低耦合**：模块内部紧密关联，模块间依赖最小化
2. **可扩展性**：支持横向扩展，便于新增功能
3. **可维护性**：代码结构清晰，易于维护和升级
4. **安全性**：采用多层安全防护机制
5. **性能优先**：优化数据库查询和缓存策略

#### 2.3.2 系统架构

采用**前后端分离架构** + **微服务思想**（单应用多模块）：

```
┌─────────────────────────────────────────────────┐
│                   前端层                         │
│              (Vue + Element UI)                  │
└─────────────────────────────────────────────────┘
                        ↓
                    Nginx反向代理
                        ↓
┌─────────────────────────────────────────────────┐
│                   后端层                         │
│              (Spring Boot + MyBatis)             │
│  ┌──────────┬──────────┬──────────┬──────────┐ │
│  │ Admin模块 │ System模块 │ Common模块 │ Framework │ │
│  └──────────┴──────────┴──────────┴──────────┘ │
└─────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────┐
│                  数据层                          │
│    ┌────────────┐     ┌────────────┐           │
│    │   MySQL    │     │   Redis    │           │
│    └────────────┘     └────────────┘           │
└─────────────────────────────────────────────────┘
```

#### 2.3.3 技术架构

**分层架构：**

| 层次 | 技术组件 | 职责 |
|-----|---------|------|
| 表现层 | Vue + Element UI | 用户界面展示 |
| 网关层 | Nginx | 反向代理、负载均衡 |
| 应用层 | Spring Boot | 业务逻辑处理 |
| 持久层 | MyBatis | 数据访问 |
| 数据层 | MySQL + Redis | 数据存储 |

#### 2.3.4 核心流程

**测试流程：**

```
用户登录 → 加载题目 → 逐题作答 → 提交答案 → 
业务层处理 → 计算分数 → 保存结果 → 返回报告
```

**评分流程：**

```
获取选项ID → 查询选项分值 → 按维度汇总 → 
查询映射表 → 确定人格类型 → 生成报告
```

### 2.6 接口设计

#### 2.6.1 RESTful API设计原则

- 使用HTTP动词（GET/POST/PUT/DELETE）
- 资源名使用名词复数
- 使用HTTP状态码表示结果
- 统一响应格式

#### 2.6.2 统一响应格式

```json
{
  "code": 200,      // 状态码
  "msg": "success", // 消息
  "data": {}        // 数据
}
```

#### 2.6.3 核心接口列表

| 接口 | 方法 | 路径 | 说明 |
|-----|------|------|------|
| 用户登录 | POST | /login | 用户登录认证 |
| 获取用户信息 | GET | /getInfo | 获取当前用户信息 |
| 分页获取题目 | GET | /questions/topic/getQuestionByPage | 分页加载题目 |
| 提交答案 | POST | /questions/topic/submit-answers | 提交测试答案 |
| 获取评分 | GET | /questions/topic/scores | 获取人格评分 |
| 保存结果 | POST | /personality/results/add | 保存测试结果 |
| 获取最近结果 | POST | /personality/results/getLatestResult | 获取最近测试结果 |
| 题目列表 | GET | /questions/topic/list | 查询题目列表 |
| 新增题目 | POST | /questions/topic | 新增题目 |
| 修改题目 | PUT | /questions/topic | 修改题目 |
| 删除题目 | DELETE | /questions/topic/{ids} | 删除题目 |

### 2.7 数据结构设计

#### 2.7.1 核心数据实体

**PersonalityResults（人格结果）**
```java
{
  resultId: Long,      // 结果ID
  userId: Long,        // 用户ID
  testDate: Date,      // 测试日期
  typeCScore: Integer, // C类型得分
  typePScore: Integer, // P类型得分
  typeMScore: Integer, // M类型得分
  typeSScore: Integer  // S类型得分
}
```

**PersonalityRecord（测试记录）**
```java
{
  recordId: Long,   // 记录ID
  userId: Long,     // 用户ID
  testDate: Date,   // 测试时间
  status: Integer   // 状态
}
```

**PersonalityAnswer（答案）**
```java
{
  id: Long,         // ID
  recordId: Long,   // 记录ID
  optionId: Long    // 选项ID
}
```

**Topic（题目）**
```java
{
  topicId: Long,        // 题目ID
  topicContent: String, // 题干内容
  prototypeId: Long     // 题目类别
}
```

#### 2.7.2 数据关系

```
用户(1) ─── (N) 测试记录
测试记录(1) ─── (N) 答案
答案(N) ─── (1) 选项
选项(N) ─── (1) 题目
测试记录(1) ─── (1) 结果
```

### 2.8 安全设计

#### 2.8.1 认证机制

- 使用JWT（JSON Web Token）进行身份认证
- Token存储在Redis中，支持过期时间配置
- Token刷新机制

#### 2.8.2 授权机制

- 基于角色的访问控制（RBAC）
- 菜单级权限控制
- 按钮级权限控制
- 数据权限控制

#### 2.8.3 数据安全

- 密码使用BCrypt加密存储
- 敏感数据传输使用HTTPS
- SQL参数化查询，防止注入
- XSS过滤

#### 2.8.4 操作审计

- 记录所有关键操作日志
- 记录登录/登出日志
- 日志不可篡改

### 2.9 性能设计

#### 2.9.1 缓存策略

- Redis缓存用户信息
- Redis缓存权限数据
- Redis缓存字典数据
- 数据库查询结果缓存

#### 2.9.2 数据库优化

- 合理使用索引
- 分页查询
- 避免N+1查询
- 连接池配置

#### 2.9.3 前端优化

- 路由懒加载
- 组件按需加载
- 图片懒加载
- CDN加速

## 5. 出错处理设计

### 5.1 出错信息

**错误码定义：**

| 错误码 | 说明 |
|-------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未授权 |
| 403 | 禁止访问 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

### 5.2 出错处理策略

1. **参数验证失败**
   - 返回400错误
   - 提示具体错误信息

2. **认证失败**
   - 返回401错误
   - 跳转到登录页

3. **权限不足**
   - 返回403错误
   - 提示无权限

4. **系统异常**
   - 返回500错误
   - 记录详细日志
   - 通知管理员

### 5.3 补救措施

1. **数据库异常**
   - 自动重试（最多3次）
   - 降级处理
   - 发送告警

2. **缓存异常**
   - 降级到数据库查询
   - 重建缓存

3. **接口超时**
   - 设置超时时间
   - 返回友好提示
   - 支持重试
