## 1. 引言

### 1.1 编写目的

本文档旨在详细说明智测心理测试系统的软件需求，明确系统的功能、性能、安全等各方面需求，为系统设计、开发、测试和验收提供依据。

### 1.2 背景

随着社会对心理健康的关注度不断提升，心理测试成为了解个人性格特征、职业倾向、心理健康状况的重要工具。传统的心理测试方式存在流程繁琐、数据分析困难、结果追溯不便等问题。

智测心理测试系统基于若依（RuoYi）v3.9.0快速开发框架，采用前后端分离架构，提供在线心理测试服务，包括人格类型测试、题目管理、结果分析等功能，为用户提供便捷、专业的心理测评体验。

### 1.3 定义

| 术语 | 定义 |
|-----|------|
| 人格类型 | 指个体在行为方式、思维方式、情绪反应等方面表现出的稳定特征模式 |
| 测试题目 | 用于评估个体心理特征的标准化问题 |
| 选项 | 题目中提供的选择项，每个选项对应特定的分值 |
| 记录 | 用户完成一次测试的完整数据 |
| 结果 | 根据用户答案计算出的各维度得分及分析报告 |
| 管理员 | 系统的超级用户，拥有所有权限 |
| 普通用户 | 系统的普通使用者，只能进行测试和查看个人结果 |

## 2. 任务概述

### 2.1 目标

**总体目标：**
构建一个功能完善、操作便捷、安全可靠的心理测试系统，为用户提供专业的心理测评服务。

**具体目标：**

1. **功能目标**
   - 实现在线人格测试功能
   - 提供完整的题目管理功能
   - 支持测试结果的自动计算和分析
   - 提供测试记录的完整追溯
   - 实现基于角色的权限管理

2. **性能目标**
   - 系统响应时间 < 2秒
   - 支持1000+并发用户
   - 系统可用性 ≥ 99.5%

3. **安全目标**
   - 用户数据加密存储
   - 操作日志完整记录
   - 权限严格控制
   - 防SQL注入、XSS等常见攻击

### 2.2 用户特点

| 用户类型 | 特点 | 技术水平 |
|---------|------|---------|
| 管理员 | 负责系统维护、题目管理、用户管理等 | 中等 |
| 测试者 | 完成心理测试，查看个人结果 | 较低 |
| 数据分析员 | 查看和分析测试数据 | 中等 |

### 2.3 假定和约束

**假定：**
- 用户具备基本的计算机操作能力
- 用户拥有稳定的网络环境
- 用户使用现代浏览器访问系统
- 数据库由专业人员进行维护

**约束：**
- 开发时间：2个月
- 开发团队：3-5人
- 预算：有限
- 技术栈：Spring Boot + Vue
- 数据库：MySQL 8.0

### 2.4 需求规定

**功能需求优先级：**

| 优先级 | 需求 |
|-------|------|
| P0 | 用户登录认证、人格测试、结果查看 |
| P1 | 题目管理、用户管理、测试记录管理 |
| P2 | 权限管理、操作日志、数据导出 |
| P3 | 系统监控、缓存管理、定时任务 |

## 3. 需求规定

### 3.1 功能需求

#### 3.1.1 用户管理（F001）

**需求描述：** 系统支持用户管理功能，管理员可以添加、修改、删除用户，用户可以修改个人信息。

**功能点：**

- F001-1：用户注册/登录
  - 支持用户名+密码登录
  - 支持验证码校验
  - 支持JWT Token认证

- F001-2：用户信息管理
  - 修改个人信息
  - 修改密码
  - 重置密码（管理员）

- F001-3：用户状态管理
  - 启用/禁用用户
  - 用户过期时间设置

**验收标准：**
- 用户可以成功登录系统
- 管理员可以创建新用户
- 用户可以修改个人信息和密码
- 禁用的用户无法登录

#### 3.1.2 权限管理（F002）

**需求描述：** 系统实现基于角色的权限控制（RBAC），不同角色拥有不同权限。

**功能点：**

- F002-1：角色管理
  - 创建、编辑、删除角色
  - 分配菜单权限
  - 分配数据权限

- F002-2：菜单管理
  - 创建多级菜单
  - 设置菜单图标
  - 配置路由地址

- F002-3：按钮权限
  - 细粒度的按钮级权限控制
  - 前端根据权限显示/隐藏按钮

**验收标准：**
- 不同角色只能访问授权的功能
- 未授权按钮自动隐藏
- 越权访问被拦截

#### 3.1.3 人格测试（F003）

**需求描述：** 用户可以在线完成人格类型测试，系统自动评分并生成分析报告。

**功能点：**

- F003-1：加载测试题目
  - 分页加载题目（每页6题）
  - 显示题目和选项
  - 支持切换上一页/下一页

- F003-2：提交答案
  - 批量提交用户选择的选项
  - 记录测试时间和用户信息
  - 生成测试记录ID

- F003-3：自动评分
  - 根据选项计算各维度得分
  - 查询映射表确定人格类型
  - 生成详细分析报告

**验收标准：**
- 用户可以逐题作答
- 系统正确计算得分
- 结果报告完整准确

#### 3.1.4 题目管理（F004）

**需求描述：** 管理员可以管理测试题目，包括增删改查。

**功能点：**

- F004-1：题目列表
  - 分页显示所有题目
  - 支持按类别筛选
  - 显示题目ID、内容、类别

- F004-2：新增题目
  - 输入题目内容
  - 选择题目类别
  - 保存到数据库

- F004-3：编辑题目
  - 修改题目内容
  - 修改题目类别
  - 保存修改

- F004-4：删除题目
  - 单条删除
  - 批量删除
  - 删除前二次确认

- F004-5：导出题目
  - 导出为Excel文件
  - 包含所有题目信息

**验收标准：**
- 管理员可以成功增删改查题目
- 导出的Excel格式正确
- 删除操作有确认提示

#### 3.1.5 选项管理（F005）

**需求描述：** 管理员可以管理题目的选项，每个选项对应不同的分值。

**功能点：**

- F005-1：选项列表
  - 显示某题目的所有选项
  - 显示选项内容和分值

- F005-2：新增选项
  - 输入选项内容
  - 设置对应分值
  - 关联到题目

- F005-3：编辑选项
  - 修改选项内容
  - 修改分值

- F005-4：删除选项

**验收标准：**
- 管理员可以成功管理选项
- 选项与题目正确关联
- 分值设置准确

#### 3.1.6 测试记录管理（F006）

**需求描述：** 系统记录所有测试记录，管理员和用户可以查看历史记录。

**功能点：**

- F006-1：个人记录查询
  - 用户查看自己的测试历史
  - 显示测试时间、状态、结果

- F006-2：全部记录查询（管理员）
  - 查看所有用户的测试记录
  - 按用户、时间筛选
  - 查看详细答题情况

- F006-3：记录详情
  - 显示用户信息
  - 显示题目和答案
  - 显示评分结果

**验收标准：**
- 用户只能查看自己的记录
- 管理员可以查看所有记录
- 记录信息完整准确

#### 3.1.7 结果管理（F007）

**需求描述：** 系统管理测试结果，包括各维度得分和人格类型分析。

**功能点：**

- F007-1：结果列表
  - 显示所有测试结果
  - 显示各维度得分
  - 显示测试日期

- F007-2：个人结果查询
  - 查询某用户的最近结果
  - 显示详细分析报告

- F007-3：结果导出
  - 导出为Excel
  - 支持批量导出

- F007-4：结果统计
  - 统计各类型人数
  - 生成统计图表

**验收标准：**
- 结果数据准确
- 导出功能正常
- 统计数据正确

#### 3.1.8 选项管理（F008）

**需求描述：** 管理测试选项的元数据。

**功能点：**

- F008-1：选项列表
- F008-2：新增选项
- F008-3：编辑选项
- F008-4：删除选项

**验收标准：**
- 选项与题目正确关联
- 分值设置准确

#### 3.1.9 系统管理（F009）

**需求描述：** 提供系统配置、日志、监控等管理功能。

**功能点：**

- F009-1：字典管理
  - 管理系统字典
  - 用于下拉框等场景

- F009-2：参数管理
  - 系统参数配置
  - 动态加载参数

- F009-3：通知公告
  - 发布系统通知
  - 管理公告列表

- F009-4：操作日志
  - 记录用户操作
  - 查询操作历史

- F009-5：登录日志
  - 记录登录信息
  - 查询登录历史

- F009-6：在线用户
  - 查看在线用户
  - 强制下线

- F009-7：定时任务
  - 管理定时任务
  - 查看执行日志

**验收标准：**
- 所有管理功能正常
- 日志记录完整
- 权限控制有效

### 3.2 性能需求

#### 3.2.1 响应时间

| 操作 | 响应时间 |
|-----|---------|
| 页面加载 | < 2秒 |
| API调用 | < 500ms |

| 结果计算 | < 3秒 |

#### 3.2.2 并发能力

- 支持1000+并发用户
- 支持100+并发测试提交
- 支持50+并发数据导出

#### 3.2.3 吞吐量

- QPS ≥ 500
- TPS ≥ 100

#### 3.2.4 资源占用

| 资源 | 占用 |
|-----|------|
| CPU | < 80%（正常负载） |

### 3.3 安全性需求

#### 3.3.1 认证与授权

- 支持JWT Token认证
- Token有效期可配置
- 支持Token刷新
- 基于角色的权限控制
- 支持数据权限控制

#### 3.3.2 数据安全

- 密码加密存储（BCrypt）
- 敏感数据加密传输（HTTPS）
- SQL注入防护
- XSS攻击防护
- CSRF防护

#### 3.3.3 审计日志

- 记录所有操作日志
- 记录登录/登出日志
- 日志不可篡改
- 日志定期备份

### 3.4 可靠性需求

| 指标 | 要求 |
|-----|------|
| 系统可用性 | ≥ 99.5% |
| 平均无故障时间（MTBF） | ≥ 720小时 |
| 平均修复时间（MTTR） | ≤ 4小时 |
| 数据备份 | 每日自动备份 |
